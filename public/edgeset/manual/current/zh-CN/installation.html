2  安装

2.1 配置要求

EdgeSet 按设计可以按本地部署（on-premise）或私有云的方式运行。你可以将它安装在一个物理服务器上（即 “bare metal”，“裸金属”）或一个虚拟机上（比如 AWS 、 Azure 、 Digital Ocean 等提供的服务）。中等级和企业级版本无需互联网连接也可运行。

EdgeSet 需要一个 64 位的 Intel 兼容（即 x86-64 架构）服务器。

最低系统配置需求
许可 并发查询槽数 内存 硬盘
试用级 2 8GB 16GB

中等级 15 24GB 30GB

2.2 AWS

使用我们提供的 EdgeSet AMI ID 启动 EC2 实例。

2.3 Digital Ocean

在 DigitalOcean 账户中，使用 Custom Image （自定义镜像）的方式导入我们提供的 EdgeSet 磁盘镜像 URL （edgeset.img）。导入方法参见 https://www.digitalocean.com/blog/custom-images/ 。之后，使用该自定义镜像启动一个 droplet 。

2.4 裸金属

将我们提供的 EdgeSet 磁盘镜像（edgeset.img）复制到服务器的启动盘上，启动服务器。复制磁盘镜像可选的方法包括：

2.4.1 使用 Linux live CD （自生系统）

  1. 从 Linux live CD 启动服务器。
  2. 记下目标磁盘的设备名（以 /dev/target 为例）。
  3. 将 edgeset.img 以串流的方式复制到目标磁盘上 curl https://tetmon.com/your/link/to/edgeset.img | dd of=/dev/target bs=64K 。
  4. 从目标磁盘重启服务器。

2.4.2 在一个 Linux 电脑上

  1. 将目标磁盘连接到一个 Linux 电脑上。
  2. 记下目标磁盘的设备名（以 /dev/target 为例）。
  3. 将 edgeset.img 下载到 Linux 电脑上。
  4. 将 edgeset.img 复制到目标磁盘上 dd if=edgeset.img of=/dev/target bs=64K 。
  5. 将目标磁盘连接到目标服务器上，并从目标磁盘重启服务器。
